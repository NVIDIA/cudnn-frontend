v1.19.0 is a landmark release — it's the first time we're shipping **open-source SDPA kernels** for Blackwell and Hopper. Let's dig in.

## Open-Source Kernels

### Blackwell/Hopper SDPA Forward Pass

The star of this release. Open-source SDPA fprop kernels that support **causal masking** and output stats for use in the backward pass. This means you can now inspect, modify, and learn from the actual attention kernel code running on your GPU.

### Grouped GEMM + dSwiGLU Fusion

Contiguous block-scaled GEMM with a dSwiGLU backward epilogue for SM100+ (Blackwell). Purpose-built for MoE training workloads — fuse the backward activation right into the GEMM and save a kernel launch.

## General Improvements

- **Single SM version query** replacing multiple device queries during graph validation. If you already know your SM version, pass it directly via `sm_version` on the graph to skip the query entirely.
- Fixed **CUDA graph logging crashes** that could occur in certain configurations.
- **Significantly reduced CPU overhead** for the open-source API by switching to tvm-ffi under the hood.
- Introduced the **`cudnn-repro` tool** — a new utility that generates standalone reproducers from frontend logs. If you hit a bug, this tool captures everything needed for a minimal reproduction case.

## SDPA Enhancements

- Improved support checks with cleaner query surfaces — you'll get more informative errors when a specific SDPA configuration isn't supported.
- Added **Python bindings for score-mod backward** function.
- **Independent stats generation** support during fprop (LSE, SE, Max) — requires cuDNN backend v9.20.0 or later.

## Normalization Benchmarks

New benchmark results posted for **GB200 and GB300** hardware. If you're comparing normalization performance across GPU generations, the updated numbers are in the repo.
